# 参考策划案对标分析（2026-03-07）

## 1. 结论（先说人话）

当前项目产出质量不达标，用户反馈完全成立。问题不是“写得慢”，而是“产出机制错误”：
- 把**巡检模板**当成了**正式策划**；
- 把**占位评分**当成了**多维评审**；
- 缺少**质量闸门**，导致低质量文档也能进入评分并出现在日报。

## 2. 量化差距（参考文档 vs 当前策划）

- 参考策划案字符数：**44,016**
- 当前策划产出字符数：**227**
- 参考二级标题数（`##`）：**23**
- 当前二级标题数：**2**
- 参考三级标题数（`###`）：**86**
- 当前三级标题数：**0**
- 参考表格行（以 `|` 行计）：**654**
- 当前表格行：**0**

判定：当前策划内容仅为巡检摘要，无法作为设计评审输入。

## 3. 根因分析

### 3.1 机制层错误
1. `planner` 任务逻辑固定生成模板文本，覆盖人工策划。
2. `player` 评分逻辑为占位实现（有文档即给固定中低分），不读取内容质量。
3. 项目经理流程只检查“有文件/有提交”，未检查“内容完整性”。

### 3.2 流程层错误
1. 缺少“文档完整度校验”门槛（字数、章节、表格、关键字段）。
2. 缺少“评分证据链”要求（每维度给分依据引用）。
3. 缺少“评分前置条件”约束（策划未达标则禁止评分）。

### 3.3 组织层错误
1. 角色职责定义正确，但执行脚本没有反映真实职责深度。
2. 监控机制偏重状态可见性，忽略了质量与可执行性。

## 4. 修复方案（立即执行）

## A. 策划产出修复
- 新增“正式策划文件”与“巡检摘要”分离：
  - 正式策划：`automation/outputs/planner/master_design.md`
  - 巡检摘要：`automation/outputs/planner/design_status.md`
- 禁止覆盖正式策划文件（只允许 append/patch，不允许模板重写）。

## B. 评分体系修复
- 玩家评分拆成 6 维（每维 10 分）：
  1) 玩法循环清晰度
  2) 数值完整度
  3) 内容深度（案件/事件）
  4) 实现可行性（Godot）
  5) 商业与版本节奏
  6) 文档可执行性（字段完整）
- 最终分 = 权重加权分（给出权重表）
- 每维必须引用策划文件章节作为证据。

## C. 质量闸门（Gate）
评分前必须满足：
1. 文档字数 >= 3000
2. 二级标题 >= 10
3. 至少包含：玩法循环、数值表、案件分层、政令清单、里程碑
4. 至少 2 个表格
未满足任一项：评分任务直接失败并输出“拒评原因”。

## D. 报告机制修复
- 项目经理日报新增：
  - `质量闸门结果`
  - `评分证据链链接`
  - `未达标项与责任角色`

## 5. 明日执行节点（可验收）

- 09:30 前：提交 `master_design.md` V1（>=3000字）
- 10:30 前：玩家输出 `score_report_v2.md`（6维评分+证据链）

## 6. 风险结论

- 当前 M1 门槛“评分 >= 8.0”没有可信度，需在评分体系修复后重新评估。
- 在修复完成前，不应再把当前评分用于里程碑决策。
