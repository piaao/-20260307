# 人工串行多角色工作流

## 目标

从现在开始，不再依赖后台定时任务驱动七角色协作，而是由主会话中的我手动、串行地扮演并执行各角色任务。

## 基本原则

1. **单线程串行**：任一时刻只激活一个角色，禁止多角色并发推进。
2. **上下文隔离**：每个角色只读取自己的上下文分区、共享基线文件和必要上游产物。
3. **输出隔离**：每个角色只写入自己的输出目录 `work/<role>/`。
4. **站会驱动**：每天以 `manual_workflow/standups/YYYY-MM-DD.md` 作为唯一任务基线。
5. **闸门优先**：策划/玩家评测结果决定是否允许下游角色继续推进。
6. **先落盘后汇报**：每次角色执行后，先更新角色输出文件，再对外汇报摘要。

## 目录约定

- 角色上下文：`manual_workflow/roles/<role>/context.md`
- 每日站会：`manual_workflow/standups/YYYY-MM-DD.md`
- 角色正式产物：`work/<role>/`

## 默认串行顺序

1. 项目经理 `pm`
2. 策划 `planner`
3. 玩家评测 `player`
4. 设计 `designer`
5. 开发 `developer`
6. 测试 `qa`
7. 音效 `sound`

## 执行节奏

- `pm`：5-10 分钟，刷新基线、风险、依赖、提交频率
- `planner`：15-25 分钟，补策划/文案/规则一致性
- `player`：10-20 分钟，读取上游产物后给出评测
- `designer/developer/qa/sound`：每个 10-25 分钟，按是否解锁决定是否执行
