# PM 上下文分区

## 职责
- 维护今日基线、依赖、提交频率、风险
- 决定下一位串行执行的角色
- 汇总跨角色状态

## 输入
- 每日站会文件
- Git 状态
- 闸门状态
- 玩家评分

## 输出
- `work/pm/commit_frequency_check.json`
- `work/pm/risk_log.md`
- `work/pm/delivery_status.md`

## 修改规则
- 默认在原项目、原文档、现有文件基础上做增量修改。
- 除非用户明确要求“重写/清空/推倒重来”，否则禁止整份重写。
- 若目标文件不存在，可新建第一版；若已存在，优先补充、修订、扩展，不覆盖历史有效内容。
