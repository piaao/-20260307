# 主视觉设计说明（县令题材）

- 更新时间：2026-03-08T13:28:00+08:00
- 责任人：设计·山青
- 对齐依据：`work/pm/daily_plan_2026-03-08.md`、`work/planner/master_design.md`

## 统一美术约束（A/B/C共用）

- 画幅标准：16:9（主KV基准 `3840x2160`，落地兼容 `1920x1080`）
- 题材边界：古代县衙写实叙事，不做玄幻化、夸张武侠化
- 核心母题：县令权责（审案、政令、民生、建设）必须同屏可读
- 指标隐喻：
  - 民心：暖色人群与生活烟火
  - 秩序：中轴/直线秩序与仪仗
  - 库银：账簿、钱箱、粮仓资产线索
  - 威望：人物站位中心性与仪式光

## 方案 A：青衙晨雾（治理开局）

- 关键词：肃穆、克制、廉明
- 场景叙述：清晨县衙正堂薄雾未散，县令入堂理政，百姓候审，书吏就位
- 色彩建议：青灰（主）+木棕（辅）+朱砂红（印章点缀）
- 构图建议：中轴对称，案台与牌匾构成稳定视觉锚点
- 适用界面：政务处理、建设规划
- 交互可读性策略：中高亮UI配色，背景对比控制在中低强度
- 风险与规避：
  - 风险：冷色过重导致情绪平
  - 规避：人物面部和权力符号处补暖光

## 方案 B：金灯夜审（案件高压）

- 关键词：紧张、冲突、戏剧张力
- 场景叙述：夜审堂内灯笼与烛火切分明暗，县令、证人、嫌犯形成三角对峙
- 色彩建议：深靛（底）+金橙（主光）+暗红（冲突警示）
- 构图建议：斜透视 + 前景遮挡（门柱/灯架），制造压迫感
- 适用界面：开堂办案、证据核验
- 交互可读性策略：UI面板使用半透明墨底与浅金描边，确保文本对比
- 风险与规避：
  - 风险：高反差导致按钮识别不稳
  - 规避：按钮层固定明度区间并保持统一阴影半径

## 方案 C：市井朝会（民生经营）

- 关键词：烟火、秩序、成长反馈
- 场景叙述：县衙外市集与公告墙同屏，百姓流动、商贸与巡防路径可见
- 色彩建议：米黄（底）+青绿（生机）+瓦褐（建筑）
- 构图建议：横向三层（前景市井/中景衙门/远景城墙）
- 适用界面：政令发布、城建升级
- 交互可读性策略：动态元素降饱和，主信息区维持高对比
- 风险与规避：
  - 风险：元素多、焦点分散
  - 规避：县令与公告墙强制为双焦点，其余转低细节层

## 方案对比结论（当前）

- A：最稳，适合首曝与系统教学场景
- B：情绪最强，最适合案件系统传播素材
- C：世界感最好，适合展示经营成长线
- 当前建议：A 作为主视觉首版，B/C 作为宣发扩展图组

## 玩法映射与视觉锚点（本轮完善）

- 政务处理（主入口）→ A 方案
  - 锚点：县衙正堂中轴、案台、牌匾
  - 信息优先级：待办政务 > 四维指标 > 氛围元素
- 开堂办案（高张力）→ B 方案
  - 锚点：县令-证人-嫌犯三角结构
  - 信息优先级：证据槽 > 判词区 > 情绪光源
- 政令发布 / 县城建设（成长反馈）→ C 方案
  - 锚点：公告墙、市集流线、城建设施
  - 信息优先级：政策收益提示 > 建筑升级反馈 > 环境氛围

## 文案与信息层级约束（本轮完善）

- 标题层：4~8 字，强调“身份 + 抉择”（如 `开堂问案`、`颁令安民`）
- 正文层：单条不超过 30 字，避免遮挡主体面部与手势
- 数值层：四维指标固定在顶部同轨道，减少方案切换跳读
- 装饰层：印章、灯笼、衙役等只做叙事补强，不承载核心交互
