- 开发引擎：Godot 4.x（GDScript）
- 首发目标：M1 完成可玩核心闭环，M2 完成垂直切片
- 本轮负责人：策划·阿席

## 一、愿景与核心卖点

1. 以“县令治理日常”为核心体验，形成政务、审案、政令、建设四系统联动。
2. 以“民心-秩序-财政-威望”四指标构成策略张力，避免单一最优解。
3. 以“官阶晋升 + 事件难度升级 + 城建解锁”形成长期动机。
4. 单局 25~35 分钟可完成一个月度循环，支持轻量重复游玩。

## 二、核心玩法循环

### 2.1 日循环（单日）
1. 政务处理：处理民生、税务、治安请求。
2. 开堂办案：完成证据核验、审理、判决。
3. 颁布政令：发布中短期策略，承担机会成本。
4. 县城建设：投入财政，换取长期增长。

### 2.2 周循环（7 日）
- 结算四维指标：民心、秩序、库银、威望。
- 触发随机事件 1~2 个（民生/治安/商贸/天灾）。
- 统计“错判率”“财政风险等级”“政令执行率”。

### 2.3 月循环（4 周）
- 进行官阶评定并解锁新事件池。
- 校验月目标完成率并发放奖励或惩罚。

## 三、官阶与解锁规则

| 官阶 | 解锁内容 | 晋升条件 |
|---|---|---|
| 九品 | 初级案件、基础政令、县衙1级 | 新手默认 |
| 八品 | 中级案件、商贸类政令、粮仓2级 | 民心>=60 且 秩序>=55 |
| 七品 | 重大案件、危机链事件、巡防营2级 | 连续两周财政为正 |

## 四、数值框架与警戒线

| 指标 | 范围 | 红线 | 说明 |
|---|---|---|---|
| 民心 | 0~100 | <30 | 低于红线触发民怨链事件 |
| 库银 | >=0 | <50 | 低于红线触发财政危机 |
| 秩序 | 0~100 | <40 | 低于红线触发治安危机 |
| 威望 | 0~100 | <20 | 晋升速度显著降低 |

### 4.1 基础自然波动（无操作）
- 民心：每日 -2
- 库银：每日 +50（基础税收）
- 秩序：每日 -1
- 威望：每日 0

### 4.2 财政风险分级
- 绿色：库银 >= 200
- 黄色：50 <= 库银 < 200
- 红色：库银 < 50（触发危机并提高失败概率）

### 5.1 审案流程标准
1. 立案：录入案由、涉事人员、争议点。
2. 证据核验：至少完成一次交叉验证。
3. 判词输出：判词含法律依据、执行动作、预期影响。
4. 社会反馈：结算民心/秩序/威望变化并写入日志。

### 5.2 案件验收规则
- 每案必须包含“触发条件 + 证据清单 + 判决后果”。
- 中高级案件必须至少包含 1 个反转分支。
- CASE-010~012 需绑定事件链 ID 以支持回放调试。

### 6.1 政令平衡目标
- 任一政令都不能在 7 日内形成“零代价净正收益”。
- 民心与秩序长期同时增长时，必须伴随财政机会成本。
- 财政修复类政令必须有 2~3 日滞后，避免无脑连点。
