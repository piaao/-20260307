# GitHub 提交频率规范（项目经理执行）

- 生效时间：2026-03-07
- 适用范围：`/home/admin/.openclaw/workspace`
- 负责人：项目经理·纪衡

## 1) 最低频率（硬规则）

1. **工作时段（09:00-23:00）每 2 小时至少 1 次提交**
2. **站会后 30 分钟内必须有 1 次提交**（当天首提）
3. **日报前（18:00）必须完成至少 1 次当日提交**
4. 若当日无有效变更，需提交 `reports/no_change_reason.md` 说明原因

## 2) 提交粒度（避免大杂烩）

- 单次提交建议控制在 3-20 个文件
- 每次提交只解决一个明确目标（策划/开发/测试/音效/流程之一）
- 禁止把“目录重构+功能改动+文档大改”混为一次提交

## 3) 提交命名规范

- `feat(role): ...` 角色功能产出
- `fix(role): ...` 缺陷修复
- `docs(role): ...` 文档更新
- `chore(pm): ...` 流程或配置调整

## 4) 自动检查（项目经理日报）

日报必须包含：
- 最近 24h 提交次数
- 最近一次提交距今分钟数
- 是否违反“2小时无提交”规则
- 风险等级（正常/警告/高风险）

## 5) 风险升级条件

- 超过 2 小时无提交：警告
- 超过 4 小时无提交：高风险（立即提醒）
- 连续 2 天未满足日报前提交：里程碑偏差
