# 后台工作任务调度策略（2026-03-08）

## 目标
- 角色任务优先于巡检任务。
- 各角色按职责 + 当日计划持续执行，直到当日任务完成。
- 总巡检仅做统一核验与汇报，不替代角色工作。

## 调度原则
1. 每个角色均使用独立 AI 定时任务（cron + agentTurn）。
2. 角色任务 prompt 必须引用当日计划文件：`work/pm/daily_plan_YYYY-MM-DD.md`。
3. 每次执行先判断“是否完成今日任务”：
   - 未完成：继续推进交付；
   - 已完成：进入质量复核/一致性检查。
4. 总巡检任务独立保留，用于统一汇报与风险升级。

## 验收要求
- 角色任务必须输出到各自 `work/<role>/` 目录。
- 项目经理必须在日报中体现：
  - 当日任务完成度
  - 提交频率状态
  - 风险等级与处置动作

## 变更规则
- 后续若调频，优先调整角色任务而非总巡检。
- 任何变更须同步更新本文件并在日报备注“策略已变更”。
